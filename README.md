# Ex Ante Fest — widget

Статичен сайт: програма, лектори, място, гид за Чепеларе, настаняване, храна.
Без бекенд — всичко се чете от JSON файлове в `/data`.

## Как да обновяваш съдържание

Всичко, което се променя, живее в `/data`. Не е нужно да пипаш HTML/CSS/JS.

- `data/chepelare-guide.json` — **трайно съдържание**, важи за всички издания:
  настаняване, къде да хапнеш. Тук добавяш нов хотел партньор, нов ресторант
  и т.н., без значение коя година е. (Полето `thingsToDo` остава във файла,
  но вече не се показва на сайта — секция „Чепеларе наоколо“ беше премахната.)
- `data/editions.json` — списък с изданията + кое е текущото (`current`).
- `data/2026/schedule.json`, `speakers.json`, `venue.json`, `partners.json`,
  `booths.json` — **специфично за изданието**: програма, лектори, място,
  партньори/спонсори, щандове и работилници. Може да се различава година от
  година.

### Пример: добавяш нов хотел партньор
Отвори `data/chepelare-guide.json`, добави запис в `accommodation`:
```json
{
  "name": "Хотел ...",
  "type": "Хотел",
  "priceRange": "от ... лв./нощувка",
  "distanceFromVenue": "... мин. пеша",
  "partner": true,
  "link": "https://... (уебсайт, по избор)",
  "mapLink": "https://www.google.com/maps/search/?api=1&query=... (по избор)",
  "notes": "..."
}
```
Полетата `link` и `mapLink` са по избор — ако ги оставиш празни (`""`), просто няма да се покаже съответният бутон. За `mapLink` най-лесно е да отвориш мястото в Google Maps, да натиснеш „Share" → „Copy link" и да го поставиш тук.

### Пример: добавяш минало издание в архива (напр. 2025 г.)
1. Направи папка `data/2025/` с `schedule.json`, `speakers.json`, `venue.json`,
   `partners.json`, `booths.json` (същия формат като `data/2026/`).
2. В `data/editions.json` добави запис в `years` за 2025 г. `current` остава `2026`.
   Готово — 2025 се появява автоматично в секция „Минали издания“ и в падащото меню.

### Пример: добавяш по-дълго описание към панел
Във `data/2026/schedule.json`, всеки запис в `events` приема поле
`description`. Ако е празно (`""`), панелът просто няма падащо меню —
ако има текст, се появява бутон „Прочети повече“ под панела:
```json
{ "time": "10:00", "title": "...", "speaker": "...", "type": "panel",
  "description": "По-дълъг текст, който се показва при клик." }
```

## Как да качваш промени (GitHub Pages)

1. Първоначално качване: качи цялата папка в нов GitHub repo → Settings →
   Pages → Deploy from branch → избери `main` / `root`.
2. За всяка следваща промяна: редактирай JSON файла директно в GitHub
   (бутон „Edit this file“ в уеб интерфейса — не е нужен терминал),
   commit-ни промяната. Сайтът се обновява автоматично за 1–2 минути.
3. Ако предпочиташ локално: `git add . && git commit -m "..." && git push`.

## Локално тестване

JSON файловете се зареждат през `fetch`, което изисква локален сървър
(не работи директно с двоен клик върху `index.html`):
```
python3 -m http.server 8000
```
после отвори `http://localhost:8000`.
